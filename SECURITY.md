# Security Policy

## No Real Patient Data

This repository is a learning and portfolio project. **Do not commit real patient data (PHI) of any kind.**

This includes but is not limited to:
- Real names, dates of birth, addresses
- Health card numbers or provincial insurance numbers
- Diagnoses, prescriptions, lab results
- Any information that could identify a real person

All sample data used in development must be synthetic (fake/generated).

## Reporting a Security Issue

If you find a security vulnerability in this project, please open a GitHub Issue tagged `security`. Do not include sensitive details in the issue - describe the nature of the issue and we will follow up privately.

## Compliance Context

This project is designed with **PIPEDA** (Canada's Personal Information Protection and Electronic Documents Act) and provincial health privacy laws in mind. In a real deployment, additional legal review and a formal Privacy Impact Assessment (PIA) would be required before handling any real patient data.
